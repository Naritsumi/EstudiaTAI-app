Tema 8: Accesibilidad, diseño universal y usabilidad

Accesibilidad universal

La accesibilidad universal, según el Artículo 2 de la Ley 51/2003, se define como:

“Condición que deben cumplir los entornos, procesos, bienes, productos y servicios, así como los objetos o instrumentos, herramientas y dispositivos, para ser comprensibles, utilizables y practicables por todas las personas en condiciones de seguridad y comodidad y de la forma más autónoma y natural posible.”

Accesibilidad en páginas Web

Una página web se considera accesible cuando sus contenidos y servicios pueden ser utilizados por todas las personas, sin importar:

  • Sus capacidades visuales, auditivas, cognitivas o motrices.
  • Las tecnologías o dispositivos que utilicen (navegadores, lectores de pantalla, etc.).

Normas WAI

La Web Accessibility Initiative (WAI) es un proyecto del World Wide Web Consortium (W3C) que desarrolla estándares para la accesibilidad web.

Principales normas WAI:

  • WCAG: Web Content Accessibility Guidelines.
  • ARIA: Accessible Rich Internet Applications.
  • UAAG: User-Agent Accessibility Guidelines.
  • ATAG: Authoring Tool Accessibility Guidelines.
  • ACT: Accessibility Conformance Testing.
  • EARL: Evaluation and Report Language.

WCAG - Web Content Accessibility Guidelines

Las WCAG establecen criterios de accesibilidad para asegurar que el contenido web (texto, imágenes, multimedia, etc.) sea accesible para todas las personas.

Principios POUR

  • Perceptible: La información y los componentes de la interfaz deben ser presentados de manera que puedan ser percibidos por todos los usuarios.
  • Operable: Los elementos interactivos deben ser utilizables a través de distintos dispositivos y métodos de interacción.
  • Comprensible: La información y el funcionamiento de la interfaz deben ser claros y fáciles de entender.
  • Robusto: El contenido debe ser compatible con diferentes tecnologías y futuras innovaciones.

Ejemplo de aplicación:

Un sitio web que utiliza texto alternativo en imágenes y etiquetas ARIA para botones cumple con el principio Perceptible y Robusto.

ARIA - Accessible Rich Internet Applications

ARIA proporciona herramientas para hacer accesibles elementos interactivos en páginas web.

  • Versión actual: 1.2.
  • Define roles para describir elementos dinámicos o complejos.
  • Facilita la comunicación con tecnologías asistivas.

ATAG - Authoring Tool Accessibility Guidelines

Guías para herramientas de creación de contenido accesible, asegurando que:

  • Las plataformas de edición permitan generar contenido accesible.
  • Sean accesibles también para los creadores con discapacidades.

Versión actual: 2.0.

UAAG - User-Agent Accessibility Guidelines

Establecen criterios para que los agentes de usuario (navegadores y reproductores multimedia) sean accesibles.

Versión actual: 2 (2.2). Existe un borrador más actualizado.

Herramientas de evaluación de accesibilidad

El Observatorio de Accesibilidad Web (OAW) es una iniciativa del Gobierno de España que asegura que los sitios web de administraciones públicas cumplan con las normativas de accesibilidad digital.

Su objetivo es promover la inclusión digital y garantizar el acceso equitativo a la información y servicios en línea para todas las personas.
