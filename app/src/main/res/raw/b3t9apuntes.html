Tema 9. Repositorios

Desarrollo Frontend

Se enfoca en la parte visible y con la que el usuario interactúa directamente de una aplicación o sitio web. Esto incluye el diseño de la interfaz, la estructura, y la interactividad de la página utilizando tecnologías como HTML, CSS y JavaScript.

  • Angular: Framework de JavaScript creado por Google para construir SPA.
  • React: Biblioteca de JavaScript desarrollada por Facebook para interfaces dinámicas.
  • Vue.js: Framework progresivo de JavaScript, conocido por su simplicidad y rendimiento.
  • Bootstrap: Biblioteca CSS para interfaces responsivas.

Desarrollo Backend

Se ocupa de la parte del servidor de una aplicación web, encargándose de la lógica de negocio, la gestión de bases de datos y la comunicación con el frontend. El backend procesa las solicitudes de los usuarios y envía respuestas adecuadas.

  • Django: Framework de Python con muchas herramientas incluidas.
  • Express.js: Framework minimalista para Node.js.
  • Spring Boot: Framework de Java para aplicaciones empresariales.
  • Ruby on Rails: Framework de Ruby enfocado en la simplicidad.
  • Flask: Microframework de Python ideal para APIs.

Concepto de repositorio

Un repositorio no es solo un lugar donde se guarda código, sino un sistema que gestiona los cambios en el código a lo largo del tiempo, permitiendo a los desarrolladores colaborar eficientemente.

Control de versiones

  • Git: Sistema distribuido, popular en GitHub, GitLab y Bitbucket.
  • Mercurial: Sistema de control de versiones escrito en Python.
  • Apache Subversion (SVN): Sistema centralizado.
  • Perforce: Utilizado en entornos empresariales y de videojuegos.

Repositorios

Repositorios Centralizados

Un servidor central almacena el código y los desarrolladores sincronizan sus cambios con él. Si el servidor falla, el acceso al código se pierde.

Ejemplos: CVS, SVN, TFS/SourceSafe.

Repositorios Distribuidos

Cada desarrollador tiene una copia completa del repositorio, incluyendo el historial de cambios.

Ejemplos: Git, Mercurial, Bazaar, Darcs, BitKeeper.

Generación de documentación

Documentar el código facilita su comprensión y uso. Herramientas como Javadoc (Java), Doxygen (C++), Sphinx (Python), JSDoc (JavaScript) y Sandcastle (.Net) generan documentación técnica.
