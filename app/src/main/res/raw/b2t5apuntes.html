Tema 5: Sistemas de gestión de bases de datos

Data Base Management System (DBMS)

Un DBMS es un software que permite la creación, gestión y manipulación de bases de datos de manera eficiente.

Existen dos grandes categorías de bases de datos:

  • Relacionales (RDBMS): Organizan los datos en tablas con filas y columnas, manteniendo relaciones entre ellas mediante claves primarias y foráneas. Ejemplo: MySQL, PostgreSQL.
  • No Relacionales (NoSQL): Utilizan modelos de datos más flexibles, como documentos, pares clave-valor, grafos o columnas. Son ideales para grandes volúmenes de datos y escalabilidad. Ejemplo: MongoDB, Apache Cassandra.

Sistemas de bases de datos relacionales (RDBMS)

  • MySQL: Muy popular en aplicaciones web, fácil de usar y de código abierto. Puerto: 3306
  • PostgreSQL: Conocida por su robustez y soporte para características avanzadas. Puerto: 5432
  • SQLite: Base de datos ligera, ideal para aplicaciones móviles y de escritorio.
  • Microsoft SQL Server: Usada comúnmente en entornos empresariales. Puerto: 1433
  • Oracle Database: Ampliamente utilizada en grandes empresas por su escalabilidad.
  • MariaDB: Un fork de MySQL con mejoras en rendimiento. Puerto: 3306

SQLite

SQLite es un sistema de gestión de bases de datos relacional muy ligero que cumple con los principios ACID (Atomicidad, Consistencia, Aislamiento y Durabilidad).

Al ser "sin servidor", no requiere configuración previa, lo que lo hace ideal para aplicaciones pequeñas.

Sistemas NoSQL

NoSQL es un término que abarca bases de datos no relacionales optimizadas para escalabilidad y flexibilidad en el almacenamiento de datos.

Tipos de bases de datos NoSQL

  • Documentales: Almacenan datos en formato JSON, BSON o XML. Ejemplo: MongoDB, CouchDB.
  • Claves-Valor: Cada entrada tiene una clave única y un valor asociado. Ejemplo: Redis, DynamoDB.
  • Columnar: Almacenan datos en columnas en lugar de filas, optimizando el rendimiento en ciertas consultas. Ejemplo: Apache Cassandra.
  • Grafos: Gestionan relaciones entre datos mediante nodos y aristas. Ejemplo: Neo4j, ArangoDB.
  • Objetos: Almacenan grandes volúmenes de datos no estructurados, como imágenes o videos. Ejemplo: ZooDB, Object Store.

Ejemplo de uso en redes sociales

Las bases de datos de grafos, como Neo4j, se utilizan para modelar relaciones en redes sociales, gestionando amigos, seguidores y likes.
