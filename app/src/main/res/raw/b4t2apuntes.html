Tema 2. Administración de bases de datos, backup y virtualización

Administradores de bases de datos

Los administradores de bases de datos (DBAs) son los encargados de usar programas y herramientas especializadas para organizar, mantener y proteger la información almacenada en una base de datos. Su trabajo es crucial para asegurar el buen funcionamiento del sistema, garantizar la seguridad de los datos y prevenir posibles fallos que afecten la integridad de la información. Algunas de las principales responsabilidades de un DBA incluyen:

  • Instalación y configuración: Se encargan de la instalación del software de gestión de bases de datos (DBMS) y su configuración adecuada según las necesidades del sistema.
  • Seguridad: Implementan medidas de seguridad para proteger los datos de accesos no autorizados, asegurándose de que solo los usuarios autorizados puedan acceder a la información. Esto incluye el control de acceso y la implementación de criptografía para proteger los datos sensibles.
  • Mantenimiento: Realizan tareas rutinarias como backups, recuperación de datos, y actualizaciones del sistema para garantizar la estabilidad y seguridad de la base de datos.
  • Optimización: Trabajan para mejorar el rendimiento del sistema de base de datos, asegurando que las consultas sean rápidas y eficientes. Esto incluye la creación de índices, la reestructuración de tablas y la configuración de parámetros de rendimiento.
  • Monitoreo: Supervisan el rendimiento del sistema de base de datos, detectando posibles fallos o caídas del sistema y asegurándose de que la base de datos esté disponible para los usuarios en todo momento.
  • Soporte: Ofrecen soporte técnico para resolver problemas relacionados con las bases de datos, como errores de consulta, problemas de acceso, entre otros.

Administrador de bases de datos de aplicaciones (ADBA)

El Administrador de Bases de Datos de Aplicaciones (ADBA) es un profesional especializado en la gestión de bases de datos para aplicaciones específicas. Mientras que el DBA tradicional se enfoca en la infraestructura y la seguridad general del sistema de bases de datos, el ADBA se concentra en las necesidades particulares de las aplicaciones que utilizan la base de datos. Algunas de las tareas clave de un ADBA son:

  • Organización de la información: Organiza la estructura de la base de datos de acuerdo con las necesidades de la aplicación específica, creando y ajustando tablas, índices y relaciones entre los datos.
  • Gestión de instantáneas: Realiza copias o "instantáneas" de los datos en momentos específicos para facilitar la recuperación de información en caso de problemas.
  • Optimización de consultas: Se enfoca en mejorar la eficiencia de las consultas a la base de datos, creando índices adecuados y ajustando los parámetros del sistema para reducir el tiempo de respuesta.

Administración de bases de datos

Conceptos básicos

La administración de bases de datos implica gestionar y mantener la integridad y la seguridad de la información. Se utilizan sistemas como MySQL, PostgreSQL, Oracle, entre otros.

Objetivo: Asegurar la correcta almacenación, acceso y protección de los datos.

Tipos de bases de datos

Se distinguen principalmente dos categorías:

  • Relacionales (SQL): Organizadas en tablas con relaciones definidas entre los datos.
  • No relacionales (NoSQL): Estructuras flexibles, ideales para grandes volúmenes y datos sin un esquema fijo.

Sistemas de almacenamiento y su virtualización

Tipos de almacenamiento

Los sistemas de almacenamiento se clasifican en:

  • DAS (Direct Attached Storage): Almacenamiento conectado directamente al servidor.
  • NAS (Network Attached Storage): Almacenamiento accesible a través de la red, ideal para compartir archivos.
  • SAN (Storage Area Network): Redes especializadas para grandes volúmenes de datos y aplicaciones críticas.

Virtualización del almacenamiento

La virtualización del almacenamiento permite gestionar de forma centralizada y abstracta distintos dispositivos, facilitando la escalabilidad y la optimización de recursos.

Ejemplo: Implementar soluciones de SDS (Software-Defined Storage) para unificar distintos discos duros en un único recurso virtual.

Políticas y procedimientos de backup y recuperación

Concepto de backup

El backup es el proceso de copiar datos para protegerlos ante posibles pérdidas o daños. Es fundamental definir políticas claras y procedimientos que aseguren la realización periódica y efectiva de copias de seguridad.

Tipos de backup:

  • Completo: Copia total de la información.
  • Incremental: Copia de los datos modificados desde el último backup.
  • Diferencial: Copia de los datos cambiados desde el último backup completo.

Recuperación de datos

La recuperación es el proceso de restaurar la información a partir de las copias de seguridad. Se recomienda realizar pruebas periódicas para verificar la eficacia de los backups y garantizar la continuidad de los servicios.

Backup de Sistemas físicos y virtuales

Backup de Sistemas Físicos

En entornos físicos, el backup se realiza sobre hardware dedicado y servidores tradicionales, utilizando herramientas específicas para gestionar estos dispositivos.

Backup de sistemas virtuales

Los entornos virtualizados requieren técnicas y herramientas que integren con plataformas de virtualización (como VMware, Hyper-V, KVM), permitiendo la realización de copias de seguridad sin afectar el rendimiento del sistema.

Visualización

La virtualización es una tecnología que permite crear versiones virtuales de recursos físicos, como servidores, almacenamiento, redes y sistemas operativos. En lugar de usar directamente el hardware físico, se crean máquinas virtuales (VMs) que permiten que varios sistemas operativos o aplicaciones corran en un solo dispositivo físico, optimizando el uso de recursos y permitiendo una mayor flexibilidad y eficiencia.

Conceptos clave

  • Virtualización: Proceso de crear representaciones virtuales de dispositivos o recursos.
  • Emulación: Capacidad de un sistema de imitar o reproducir el comportamiento de otro sistema.

Tipos de archivos y extensiones

  • .vmx: Configuración de máquinas virtuales en VMware.
  • .vdi: Formato de disco virtual utilizado por Oracle VirtualBox.
  • .vbox: Configuración de máquinas virtuales en VirtualBox.
  • .ova: Archivo empaquetado que contiene una VM completa.
  • .vmdk: Formato de disco duro virtual de VMware.

Virtualización de máquinas

  • Máquina virtual (VM): Entorno que simula un sistema operativo completo.
  • Hipervisor: Permite la coexistencia de múltiples VMs en un solo servidor físico.

Plataformas

  • IaaS: Infraestructura como servicio. Ejemplo: AWS, Azure.
  • PaaS: Plataforma como servicio. Ejemplo: Heroku.
  • SaaS: Software como servicio. Ejemplo: Gmail.

Docker

Docker es una plataforma de código abierto diseñada para facilitar la creación, el despliegue y la ejecución de aplicaciones utilizando contenedores.

  • Imagen Docker: Contiene herramientas, librerías y la aplicación en sí.
  • Docker Engine: Gestiona imágenes y contenedores.
  • Docker Client: Herramienta de línea de comandos para Docker.

Microservicios

Arquitectura de software que estructura una aplicación como un conjunto de servicios pequeños e independientes.

Herramientas y tecnologías

Entre las herramientas más utilizadas en este ámbito se encuentran:

  • SGBD: MySQL, PostgreSQL, Oracle y SQL Server.
  • Virtualización: VMware, Hyper-V, KVM y Xen.
  • Software de backup: Veeam, Acronis, Bacula y otros.
  • SDS: Soluciones de Software-Defined Storage para la virtualización del almacenamiento.
