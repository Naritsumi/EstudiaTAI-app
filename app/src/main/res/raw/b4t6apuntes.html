Tema 6. Medios de comunicación

Los dispositivos de interconexión de redes son fundamentales para comunicar ordenadores y otros equipos dentro de una red. Permiten ampliar la cobertura, mejorar el rendimiento y garantizar la correcta transmisión de los datos entre diferentes segmentos o tipos de redes.

Dominios en redes

Dominio de colisión

Es un segmento de red donde pueden ocurrir colisiones de paquetes. Todos los dispositivos dentro del mismo dominio comparten el medio de transmisión y pueden interferir entre sí cuando intentan enviar datos al mismo tiempo.

Dominio de difusión (Broadcast Domain)

Es la parte de la red en la que cualquier paquete de difusión (broadcast) enviado por un dispositivo es recibido por todos los demás dentro de ese dominio.

1. Repetidores y concentradores (Nivel 1 OSI)

Repetidores

Se utilizan para amplificar o regenerar señales en una red, extendiendo la distancia sobre la cual una señal puede viajar sin degradarse.

Hubs (Concentradores)

Dispositivo que conecta múltiples equipos en una red local (LAN) y transmite los datos recibidos a todos los puertos. Todos los dispositivos comparten un único dominio de colisión.

Tipos de Hubs

  • Pasivos: solo actúan como punto de conexión; no amplifican la señal.
  • Activos: regeneran y amplifican las señales antes de retransmitirlas.
  • Inteligentes: permiten monitorización, control y segmentación de red (3ª generación con base RISC).
Desventajas principales:
  • Ancho de banda compartido: todo el tráfico se transmite a todos los puertos, provocando congestión.
  • Seguridad: los datos pueden ser interceptados fácilmente por otros dispositivos.

2. Bridges (Puentes) – Nivel 2 OSI

Un bridge conecta dos segmentos de red y filtra el tráfico, reduciendo colisiones y mejorando el rendimiento. Permite la comunicación entre redes locales separadas.

  • Separan dominios de colisión, pero no de difusión.
  • Permiten conectar distintos medios físicos.

Ventajas

  • Interoperabilidad entre niveles 1 y 2 del modelo OSI.
  • Fiabilidad: aislamiento de fallos.
  • Eficiencia: menor congestión.
  • Seguridad: segmentación del tráfico.
Desventajas:
  • No filtran difusión (broadcast).
  • Ineficientes a gran escala por sobrecarga administrativa.
  • Problemas de temporización si se encadenan muchos puentes.

Tareas básicas

  • Aprender direcciones MAC por interfaz.
  • Filtrar tramas según dirección de destino.
  • Reenviar tramas a redes remotas si el destino es desconocido o de grupo.

Tipos de puentes

  • Simple: conecta dos segmentos de red.
  • Local: conecta segmentos dentro del mismo edificio.
  • Remoto: conecta redes en distintas ubicaciones mediante WAN.
  • Inalámbrico: enlaza segmentos mediante conexión Wi-Fi.

3. Switches (Conmutadores) – Nivel 2 OSI

Los switches conectan dispositivos dentro de una red y utilizan direcciones MAC para enviar los datos al puerto correcto. Cada puerto es un dominio de colisión independiente.

  • Separan dominios de colisión.
  • No separan dominios de difusión.
  • Permiten velocidad nominal por puerto (10/100/1000 Mbps).

Técnicas de conmutación

  • Store-and-Forward: almacena y verifica CRC antes de reenviar (obligatorio si hay distintas velocidades).
  • Cut-Through: reenvía tan pronto se lee la dirección de destino (baja latencia).
  • Fragment-Free: lee los primeros 64 bytes antes de reenviar (reduce errores).
  • Adaptive Switching: combina las técnicas anteriores según las condiciones de red.

Facilidades adicionales

  • Full-duplex y control de flujo.
  • Autonegociación de velocidad y modo.
  • Agregación de enlaces para mayor rendimiento.
  • Soporte VLAN (802.1q) y puertos trunk.
  • Integración con routing para redes complejas.

4. Routers – Nivel 3 OSI

Los routers dirigen el tráfico entre diferentes redes, utilizando direcciones IP para determinar la mejor ruta hacia el destino.

Clasificación

  • Locales: interconectan redes directamente conectadas.
  • De área extensa: enlazan redes distantes (WAN).

Actualización de tablas

  • Estáticas: configuradas manualmente.
  • Dinámicas: se actualizan automáticamente mediante protocolos de enrutamiento.

Otras variantes

  • Multiprotocolo: soporta varios protocolos de capa 3 simultáneamente.
  • Brouter: combina funciones de bridge y router.
  • Switching Router: combina conmutación y enrutamiento.
  • Trouter: une funcionalidades de enrutador y conmutador (LAN y WAN).
  • VRF (Virtual Routing and Forwarding): crea routers lógicos independientes para mayor seguridad.

5. Gateways (Pasarelas) – Nivel 4 OSI en adelante

Los gateways conectan redes que utilizan diferentes protocolos, actuando como traductores entre sistemas.

Ventajas

  • Simplifican la gestión de red.
  • Permiten conversión de protocolos.
Desventajas:
  • Alto costo.
  • Ralentización por conversión de protocolos.

6. Firewalls

Los firewalls controlan el tráfico entrante y saliente según reglas de seguridad. Protegen la red frente a accesos no autorizados.

Tecnologías de firewall

  • Filtrado de paquetes.
  • A nivel de circuito: con tablas de estado estáticas.
  • A nivel de aplicación: como los proxies.
  • Inspección de estados: registra el estado de las conexiones para detectar ataques.

7. Proxies

Un proxy actúa como intermediario entre cliente y servidor, gestionando las solicitudes y respuestas. Permite controlar el tráfico y mejorar la seguridad.

  • Seguridad: filtra contenidos y usuarios.
  • NAT: traduce direcciones para el acceso a Internet.
  • Caché: almacena contenido para acelerar respuestas.

Proxy Inverso

Se sitúa entre los clientes y los servidores backend, mejorando la seguridad y gestionando las peticiones antes de que lleguen al servidor principal.

8 Tipos de Redes de Conmutación

Las redes de conmutación definen cómo se transmiten los datos entre los nodos de una red. Según la técnica utilizada, pueden organizar el envío de la información en circuitos, paquetes, mensajes o mediante difusión.

8.1. De Conmutación de Circuitos

Tipo de red de telecomunicaciones en la que se establece un canal o circuito dedicado entre los puntos de comunicación durante toda la sesión.

  • Los recursos se reservan durante toda la sesión, aunque esto reduce la eficiencia.
  • No se aprovecha totalmente el ancho de banda disponible.
  • Proporciona una calidad de servicio (QoS) constante y predecible.

Ejemplos de uso

  • RTB: Red Telefónica Básica.
  • RDSI: Red Digital de Servicios Integrados.

8.2. De Conmutación de Paquetes

En este tipo de red, los datos se dividen en paquetes más pequeños antes de ser enviados. Cada paquete puede tomar diferentes rutas hasta llegar a su destino.

  • Uso eficiente de los recursos de la red.
  • Tolerante a fallos: si una ruta se congestiona o falla, los paquetes pueden ser redirigidos.

Según el modo de entrega

  • Entrega sin conexión (Ejemplo: IP - Datagrama)
    • Cada paquete se trata de forma independiente y puede seguir rutas distintas.
    • No se garantiza el orden de entrega.
    • Cada paquete contiene la dirección de destino.
  • Entrega basada en conexión (Ejemplo: TCP - Circuitos virtuales)
    • Se establece una conexión lógica antes de transmitir los paquetes.
    • Minimiza el desperdicio de red, el enlace es lógico.
    • Tipos:
      • PVC (Permanent Virtual Circuit): circuito virtual permanente.
      • SVC (Switched Virtual Circuit): circuito virtual conmutado.
    • Ejemplos de tecnologías: ATM, Frame Relay, X.25, MPLS.
Frame Relay y X.25 son tecnologías de red orientadas a conexión, utilizadas históricamente para comunicaciones WAN, que dividen los datos en paquetes (tramas) y los transmiten mediante circuitos virtuales.

8.3. Redes de Conmutación de Mensajes

En este tipo de redes, cada mensaje completo se transfiere de un nodo a otro hasta llegar al destino final.

  • Cada nodo almacena el mensaje completo antes de enviarlo al siguiente (técnica store-and-forward).
  • Puede introducir retardos, pero garantiza la recepción del mensaje completo.

8.4. Redes de Difusión (Broadcast)

En estas redes, un único emisor transmite datos a todos los dispositivos conectados. No se requiere una conexión directa entre el emisor y cada receptor, ya que comparten el medio de transmisión.

Ejemplos de redes de difusión

  • LAN (Local Area Network): redes de área local con tecnología Ethernet.
  • WLAN (Wireless LAN): redes inalámbricas como Wi-Fi.
  • Redes Satelitales: un emisor transmite a múltiples receptores por cobertura satelital.
Wi-Fi y 5G son ejemplos modernos de redes inalámbricas que emplean técnicas de difusión y multiplexación avanzada para ofrecer conectividad de alta velocidad y baja latencia.
